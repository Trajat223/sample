<h2>Project Description:</h2>

This project enables laptop administrators to define, schedule, and execute custom workflows through scripts on endpoints managed via IBM MaaS360, then capture and review execution status and logs directly within the MaaS360 portal.


<h2>Justification:</h2>

While MaaS360 provides robust built‑in administrative actions, it does not natively support author‑defined workflows or script execution with result capture in the portal. Organizations need a flexible way to tailor device management to specialized requirements; without this, admins face gaps in automation, visibility, and compliance reporting for laptop operations.


<h2>Scope:</h2>
<ul>
<li>Authoring administrative scripts by IT administrators to perform device management tasks such as scheduled restarts, temporary file cleanup, or system maintenance.</li>

<li>Uploading and distributing scripts through the MaaS360 management console and assigning them to defined groups or individual target devices.</li>

<li>Configuration of recurring execution schedules, including start time, frequency, and recurrence intervals, as defined by administrative policy.</li>

<li>Secure delivery of scheduled task assignments from the MaaS360 server to enrolled and managed devices.</li>

<li>Autonomous execution of assigned scripts on target devices according to the configured schedule, without requiring user intervention.</li>

<li>Collection and reporting of execution results, including success or failure status, standard output, error logs, and timestamps, back to the MaaS360 management server.</li>

<li>Administrative visibility into execution history through the MaaS360 dashboard, enabling auditing, compliance verification, and troubleshooting.</li>
</ul>
